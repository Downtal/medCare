1. Quy trình Cốt lõi (Core Workflow)
Nhóm lệnh dùng nhiều nhất để xây dựng tính năng từ A đến Z.
Lệnh	Ý nghĩa & Tác dụng
/gsd-new-project	Khởi tạo dự án/lộ trình làm việc mới.
/gsd-discuss-phase [N]	Thảo luận để chốt phương án, UI/UX, logic trước khi lập kế hoạch cho Phase N.
/gsd-plan-phase [N]	Phân tích và tạo file kế hoạch code chi tiết (XML) cho Phase N.
/gsd-execute-phase [N]	AI bắt đầu viết code dựa trên kế hoạch và tự động Git Commit.
/gsd-verify-work [N]	Nghiệm thu tính năng, nếu có bug AI sẽ tự động phân tích và fix.
/gsd-ship [N]	Tạo Pull Request (PR) đóng gói code của Phase vừa xong.
/gsd-next	Tự động nhận diện trạng thái và chạy bước tiếp theo (Rất nhàn).
/gsd-complete-milestone	Đóng gói cột mốc, lưu trữ tiến độ khi đã xong toàn bộ Lộ trình.
/gsd-new-milestone	Khởi tạo cột mốc (phiên bản) tiếp theo cho dự án.
________________________________________
2. Làm việc với Dự án Cũ (Brownfield)
Nhóm lệnh dùng khi đưa GSD vào dự án đã có sẵn code (như MedCare của bạn).
Lệnh	Ý nghĩa & Tác dụng
/gsd-map-codebase	Quét toàn bộ mã nguồn cũ để học cấu trúc, thư viện, API.
/gsd-ingest-docs [dir]	Quét các file tài liệu thiết kế có sẵn (PRD, Spec) để đưa vào dữ liệu của AI.
________________________________________
3. Sửa nhanh & Code Nháp (Quick Tasks & Spikes)
Dùng cho các công việc không cần đưa vào lộ trình dài hạn.
Lệnh	Ý nghĩa & Tác dụng
/gsd-quick	Làm một task nhỏ nhanh chóng (VD: Thêm 1 nút bấm). Có thể kèm cờ --discuss để thảo luận.
/gsd-fast <văn_bản>	Làm task siêu tốc bỏ qua mọi bước lên kế hoạch.
/gsd-spike	Code nháp nghiệm thu nhanh nghiệm (chạy thử nghiệm tính khả thi trước khi làm thật).
/gsd-sketch	Thiết kế nhanh 2-3 biến thể giao diện HTML để chọn nghiệm trước.
/gsd-spike-wrap-up	Đóng gói kết quả của lệnh spike thành "kỹ năng" để AI áp dụng cho dự án sau này.
________________________________________
4. Quản lý Lộ trình (Phase Management)
Chỉnh sửa Roadmap (Lộ trình) khi đang làm dở.
Lệnh	Ý nghĩa & Tác dụng
/gsd-add-phase	Thêm một Phase mới vào cuối lộ trình hiện tại.
/gsd-insert-phase [N]	Chèn khẩn cấp một Phase vào giữa lộ trình (Ví dụ sếp bắt thêm tính năng gấp).
/gsd-remove-phase [N]	Xóa bỏ một Phase, hệ thống tự đánh số lại.
/gsd-list-phase-assumptions	Xem trước AI định dùng phương án gì để code Phase này.
________________________________________
5. Quản lý UI/UX & Code Quality (Chất lượng)
Lệnh	Ý nghĩa & Tác dụng
/gsd-ui-phase	Khởi tạo bộ luật thiết kế UI chuẩn (UI-SPEC.md) cho dự án frontend.
/gsd-ui-review	Bắt AI audit/review lại code xem có đúng chuẩn thiết kế UI/UX không.
/gsd-review	Yêu cầu AI kiểm tra chéo (Peer review) nhánh code hiện tại.
/gsd-secure-phase [N]	Ép kiểm tra bảo mật nghiêm ngặt cho một Phase.
________________________________________
6. Theo dõi & Điều hướng (Navigation & Utils)
Lệnh	Ý nghĩa & Tác dụng
/gsd-progress	Xem mình đang làm đến bước nào, task nào.
/gsd-settings	Mở bảng cài đặt của GSD (Chỉnh sửa quy trình, cấu hình AI).
/gsd-manager	Mở bảng điều khiển tương tác TUI (Menu lựa chọn bằng phím mũi tên).
/gsd-debug	Khởi chạy luồng fix bug có hệ thống.
/gsd-help	Liệt kê danh sách các lệnh ngay trong terminal.
/gsd-update	Cập nhật GSD lên phiên bản mới nhất.
________________________________________
7. Quản lý Phiên làm việc (Session & Backlog)
Giúp bạn quản lý thời gian nghỉ ngơi và ghi nhớ ý tưởng.
Lệnh	Ý nghĩa & Tác dụng
/gsd-pause-work	Tạo file bàn giao (Handoff) để lưu trạng thái trước khi tắt máy tính.
/gsd-resume-work	Khôi phục trạng thái làm việc từ session trước.
/gsd-add-todo	Thêm ghi chú công việc cần làm rải rác.
/gsd-add-backlog	Ghi chú ý tưởng lớn nhưng chưa làm ngay vào kho chứa (Parking lot).
/gsd-thread	Tạo "Luồng" để lưu trữ ngữ cảnh liên kết xuyên suốt nhiều ngày.
