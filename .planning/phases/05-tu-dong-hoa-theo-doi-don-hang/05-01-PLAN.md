## Goal
Tạo Endpoint để nhận Webhook từ GHN, xác thực bằng Custom Header Token, cập nhật trạng thái bảng `shipments` và gọi sang `order-service`.

## Tasks

- Thêm `ghn.webhook.secret-key` vào `.env` (giả lập) và load vào `application.yml`.
- Mapping vào `GHNConfig.java`.
</action>

<task>
<action>
Cập nhật `ShippingService.java` để xử lý Webhook.
- Viết hàm `processWebhook(GHNWebhookRequest request, String token)`.
- Kiểm tra `token` có khớp với `webhookSecretKey` không. Nếu không, ném Exception.
- Map trạng thái của GHN:
  - `ready_to_pick`, `picking`, `storing`, `delivering` -> `SHIPPING`
  - `delivered` -> `DELIVERED`
  - `return`, `returned` -> `RETURNED`
- Cập nhật entity `Shipment`.
- Gọi `OrderClient` để báo cho `order-service`.
</action>

- BE/shipping-service/src/main/java/com/medcare/shippingservice/service/ShippingService.java

</read_first>
<acceptance_criteria>

- Cập nhật thành công DB và gọi sang `order-service`.
</acceptance_criteria>
</task>

<task>
