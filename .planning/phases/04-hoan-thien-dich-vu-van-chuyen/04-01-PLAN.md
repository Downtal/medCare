## Goal
Tích hợp API của GHN để tự động tạo mã vận đơn (Tracking Code) khi đơn hàng thanh toán thành công hoặc được xác nhận (đối với COD).

## Tasks

<task>
<action>
Thêm thông số cấu hình GHN vào `shipping-service/application.yml` và tạo `GHNConfig.java`.
- `ghn.api-url`, `ghn.token`, `ghn.shop-id`.
</action>
<acceptance_criteria>
- File cấu hình đầy đủ thông tin để kết nối GHN Sandbox.
</acceptance_criteria>
</task>

<task>
<action>
Tạo các DTO `GHNCreateOrderRequest.java` và `GHNCreateOrderResponse.java`.
- Map đúng các trường yêu cầu của GHN: `to_name`, `to_phone`, `to_address`, `to_ward_code`, `to_district_id`, `weight`, `items`.
</action>
<acceptance_criteria>
- DTO chứa đúng format JSON mà GHN API yêu cầu.
</acceptance_criteria>
</task>

<task>
<action>
Xây dựng `ShippingService.java`.
- Phương thức `createShippingOrder(ShippingRequest request)` sử dụng `RestTemplate` hoặc `WebClient` (hoặc `OpenFeign` gọi ra ngoài) để POST dữ liệu lên GHN.
- Xử lý phản hồi, lấy `order_code` từ GHN và lưu vào bảng `shipments`.
</action>
<acceptance_criteria>
- Phương thức trả về đối tượng Shipment chứa Tracking Code hợp lệ.
</acceptance_criteria>
</task>

<task>
<action>
Xây dựng Endpoint nội bộ trong `ShippingController.java`.
- `POST /api/shipping/internal/create`
</action>
<acceptance_criteria>
- Endpoint hoạt động và nhận request từ `order-service`.
</acceptance_criteria>
</task>

<task>
<action>
Cập nhật `order-service` để gọi `shipping-service`.
- Tạo `ShippingClient` bằng OpenFeign.
- Cập nhật logic trong `OrderService.updatePaymentStatusByCode` và logic xử lý đơn hàng COD để tự động gọi `ShippingClient.createOrder()`.
</action>
<acceptance_criteria>
- Đơn hàng sau khi PAID (hoặc CONFIRMED với COD) sẽ có thông tin vận chuyển bên `shipping-service`.
</acceptance_criteria>
</task>
